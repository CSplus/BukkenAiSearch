# Codex開発ルール

## プロジェクト概要

Bubble.ioで構築する不動産物件AI検索システム

## 開発方針

- Bubble.ioを前提とする
- 推測で仕様を決めない
- 不明点は質問する
- データベース設計を重視する
- 保守性を優先する

## 命名規則

### Data Type

- Property
- Customer
- Inquiry
- Area

### Field

- title
- rent
- area
- address
- description

## 実装前のルール

実装前に

- 要件確認
- データ設計
- API設計

を提示すること。

## 禁止事項

- APIキーを保存しない
- パスワードを保存しない
- 推測で機能追加しない
